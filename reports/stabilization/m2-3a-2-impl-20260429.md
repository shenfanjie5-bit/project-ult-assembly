# M2.3a-2 — Phase 1 Default-Wire Fix (Cross-Module Implementation)

## Status

**M2.3a-2 status: COMPLETE (review-folded).** Five new commits across four
module repositories land the Phase 1 wiring scaffolding identified in the
M2.3a-1 review and fully scoped by the M2.3a-0 design audit. The
orchestrator's `_default_graph_phase1_provider()` no longer hits a hidden
fail-closed substitution at asset-evaluation time; it now resolves the
runtime via graph-engine's `build_graph_phase1_runtime_from_env()` and
falls through to an explicit fail-closed provider when either graph-engine
is not installed or env vars are absent.

## Files changed (5 repos / 5 commits)

### data-platform (branch `m2-3a-2-phase1-adapters`, commit `1655488`)

**New module:** `src/data_platform/cycle/graph_phase1_adapters.py`

* `PostgresCandidateDeltaReader` — real implementation. Queries
  `cycle_candidate_selection JOIN candidate_queue WHERE
  payload_type='Ex-3' AND validation_status='accepted' ORDER BY
  ingest_seq`, deserialises payload JSONB into
  `Ex3CandidateGraphDelta` (= `contracts.schemas.CandidateGraphDelta`).
  Resilient to either dict or stringified JSON payloads (psycopg version
  variance).
* `IcebergEntityAnchorReader` — real implementation. Reads
  `canonical_entity` / `entity_alias` Iceberg tables via
  `data_platform.serving.reader.read_canonical`. Falls back to empty
  results on `CanonicalTableNotFound` so first-cycle warehouses without
  these tables yet don't block Phase 0. Lives in data-platform (not
  entity-registry) per data-ownership: data-platform CLAUDE.md OWNs the
  storage of these tables; entity-registry owns the canonical id
  *generation* rules.
* `StubCanonicalGraphWriter` — type-correct stub. Raises
  `NotImplementedError` on actual `write_canonical_records`. **M2.6
  follow-up:** define a `canonical.graph_*` Iceberg table family and
  implement the real promotion write-back.

**New tests:** `tests/cycle/test_graph_phase1_adapters.py` — 11 unit
tests (typed delta deserialisation, JSON-string fallback, anchor
mapping, entity existence filtering, missing-table fallback,
empty-input short-circuits, stub writer NotImplementedError, factories).

### main-core (branch `m2-3a-2-regime-reader`, commit `f70d01b`)

### graph-engine (branch `m2-3a-2-phase1-runtime-from-env`, commit `d271bae`)

**Modified:** `graph_engine/providers/phase1.py`

* New public factory `build_graph_phase1_runtime_from_env()` composing
  the 7 `GraphPhase1Service` constructor params from env (Neo4j config,
  `DATABASE_URL` for status store, `GRAPH_PHASE1_SNAPSHOT_ARTIFACT_ROOT`
  for snapshot writer) plus 4 cross-module adapters via lazy imports.
  Each adapter has a keyword-only override for test seams. Lazy imports
  surface `ImportError` as `EnvironmentError` so callers can use a single
  `except EnvironmentError` for the fall-back.
* `build_graph_phase1_provider(runtime=None)` — when `runtime` is None,
  calls `build_graph_phase1_runtime_from_env()` instead of substituting
  fail-closed internally. EnvironmentError propagates to caller.
* `GraphPhase1AssetFactoryProvider.get_resources()` — Phase 0 pattern
  migration: raises `RuntimeError("Graph Phase 1 runtime dependencies
  are not configured")` when `runtime` is None, instead of silently
  substituting `_FailClosedGraphPhase1Runtime()`. Misconfigurations now
  surface at Definitions-load time, not asset-evaluation time. Order
  reversed so the runtime check runs before `_require_dagster()` for a
  more actionable error message.

**Modified:** `graph_engine/providers/__init__.py` — exports
`build_graph_phase1_runtime_from_env`.

**Updated test:** `tests/unit/test_phase1_provider.py:148` — passes a
sentinel runtime to `build_graph_phase1_provider()` since the no-arg
version now triggers env-driven construction (which would fail in unit
test venvs).

**New tests:** `tests/unit/test_phase1_from_env.py` — 6 tests (full
override path returning real `GraphPhase1Service`, three EnvironmentError
fail paths, explicit snapshot_writer override, get_resources raises on
unset runtime).

**Test sweep:** **421 passed / 21 skipped / 0 failed** (was 415/21
post-M2.3a-1 — net +6 tests, no regressions).

## Review fold-in (P1 + P2 + P3 + HIGH test gaps closed)

The post-implementation review pass landed the following changes inside
the same commits (graph-engine `32438a4`, data-platform `cfeeb2f`,
orchestrator `0ccae67`):

**Code (graph-engine):**
- New public factory `build_fail_closed_graph_phase1_provider()` in
  `phase1.py` so callers (orchestrator) no longer have to import the
  private `_FailClosedGraphPhase1Runtime` class across module boundaries
  (P2, code reviewer + boundary).
- `build_graph_phase1_runtime_from_env` now consolidates the 3
  data-platform adapter imports into a single try/except block (more
  concise + same-or-fail semantics) and wraps each `from_env()` call in
  an additional `(ValueError, RuntimeError)` catch so future env-aware
  adapter constructors that raise validation errors still surface as
  `EnvironmentError` per the docstring contract (P2, code reviewer).

- Added `_validated_identifier()` helper + module-load-time validation
  of `CANDIDATE_QUEUE_TABLE` and `CYCLE_CANDIDATE_SELECTION_TABLE`
  before any SQL f-string interpolation. Defence in depth against future
  changes that expose these names to env / config (P1, code reviewer).
- Thread-safe lazy engine initialisation with `threading.Lock` in
  `PostgresCandidateDeltaReader._resolve_engine` (P2, code reviewer).

**Code (orchestrator):**
- `_build_fail_closed_graph_phase1_provider` now calls graph-engine's
  public `build_fail_closed_graph_phase1_provider()` factory instead of
  importing private `_FailClosedGraphPhase1Runtime` (P2, code reviewer +
  CLAUDE.md compliance).

**Tests (graph-engine, +4 new):**
