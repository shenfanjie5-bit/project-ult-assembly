## Module ownership decision (Option A vs Option B)

### Option A: Fix in graph-engine

graph-engine adds public env-factories that orchestrator calls. Compatible
with both `orchestrator/CLAUDE.md` ("不拥有业务实现") and
`graph-engine/CLAUDE.md` ("唯一负责图谱计算 + Status Guard").

### Option B: Fix in orchestrator

orchestrator imports adapters from data-platform + graph-engine and wires
them. This **violates** orchestrator/CLAUDE.md "下游模块不得反向依赖
orchestrator"... wait actually that's wrong direction — Option B has
orchestrator importing from data-platform/graph-engine which is *forward*
not reverse. But it does push wiring complexity into orchestrator, which
borders on "业务实现".

### Decision: Option A for both Phase 0 and Phase 1

**Rationale:**
- graph-engine `CLAUDE.md` explicitly requires "Status Guard" (validate `graph_status == ready` before reading Neo4j) — that's graph-engine's responsibility to enforce in its own factory.
- Phase 0 fits cleanly because graph-engine already owns Neo4jClient + GraphStatusManager.
- Phase 1 is harder because of cross-module adapter deps, but if data-platform / main-core / entity-registry expose `from_env()` factories, graph-engine composes them.
- Orchestrator stays a thin sequencing layer.

**Caveat for Phase 1:** Cross-module env-factory APIs may need to be added in
data-platform / main-core / entity-registry. That's a coordination cost that
wasn't visible at M2.0 audit time. **Recommend separate sub-round M2.3a-2
to land Phase 1 fix.**

1. **New file:** `graph-engine/graph_engine/providers/phase0.py`

   ```python
   from typing import Protocol
   from graph_engine.client import Neo4jClient, load_config_from_env
   from graph_engine.status import (
       GraphStatusManager,
       PostgreSQLStatusStore,
       check_live_graph_consistency,
   )

   class GraphPhase0StatusRuntime(Protocol):
       """Phase 0 graph-status runtime boundary (matches orchestrator
       GraphStatusProvider Protocol)."""

       def get_graph_status(
           self, *, candidate_freeze: object, cycle_id: str
       ) -> object: ...


   class Neo4jGraphStatusProvider:
       """Real impl: reads neo4j_graph_status row + (optionally) verifies
       live graph consistency before returning to orchestrator."""

       def __init__(
           self,
           *,
           status_manager: GraphStatusManager,
           client: Neo4jClient | None = None,  # for optional consistency probe
       ) -> None:
           self._status_manager = status_manager
           self._client = client

       def get_graph_status(
           self, *, candidate_freeze: object, cycle_id: str
       ) -> dict:
           status = self._status_manager.get_status()
           # Return a plain dict; orchestrator's _graph_status_ready
           # consumes either Mapping or object via duck-typing.
           return {
               "graph_status": status.graph_status,
               "graph_generation_id": status.graph_generation_id,
               "node_count": status.node_count,
               "edge_count": status.edge_count,
               "key_label_counts": status.key_label_counts,
               "checksum": status.checksum,
               "writer_lock_token": status.writer_lock_token,
               "last_verified_at": (
                   status.last_verified_at.isoformat()
                   if status.last_verified_at else None
               ),
               "last_reload_at": (
                   status.last_reload_at.isoformat()
                   if status.last_reload_at else None
               ),
               "cycle_id": cycle_id,  # echo for traceability
           }


   class _FailClosedGraphPhase0StatusRuntime:
       def get_graph_status(self, *, candidate_freeze, cycle_id) -> object:
           raise RuntimeError(
               "Graph Phase 0 status runtime is not configured; provide a "
               "Neo4j-backed runtime via build_graph_phase0_status_provider()."
           )


   def build_graph_phase0_status_runtime_from_env() -> GraphPhase0StatusRuntime:
       """Construct a real runtime from env vars: NEO4J_URI/USER/PASSWORD/DATABASE
       + DATABASE_URL (for status store).

       Raises EnvironmentError if required env vars missing.
       """
       try:
           neo4j_config = load_config_from_env()
       except KeyError as exc:
           raise EnvironmentError(
               f"Phase 0 graph status requires Neo4j env: {exc}"
           ) from exc

       try:
           store = PostgreSQLStatusStore.from_database_url()
       except (KeyError, RuntimeError) as exc:
           raise EnvironmentError(
               f"Phase 0 graph status requires DATABASE_URL: {exc}"
           ) from exc

       status_manager = GraphStatusManager(store)
       client = Neo4jClient(neo4j_config)
       return Neo4jGraphStatusProvider(
           status_manager=status_manager, client=client
       )


   def build_graph_phase0_status_provider(
       runtime: GraphPhase0StatusRuntime | None = None,
   ) -> GraphPhase0StatusRuntime:
       """Public factory. Falls back to env-based construction.

       Returns a runtime that satisfies orchestrator's GraphStatusProvider
       Protocol. If env vars unset OR runtime construction fails,
       the caller is responsible for handling — caller may choose to use
       _FailClosedGraphPhase0StatusRuntime() explicitly for fail-closed
       evidence-friendly behavior.
       """
       if runtime is not None:
           return runtime
       return build_graph_phase0_status_runtime_from_env()
   ```

## Test strategy

### Live integration tests (require compose stack)

- Pattern: skip with `pytest.mark.skipif(os.getenv("NEO4J_PASSWORD") is None, ...)`
- Use `Neo4jClient(load_config_from_env()).verify_connectivity()` for double-check
- Use `SchemaManager(client).apply_schema()` to ensure constraints
- For status tests: write directly to PG via `PostgreSQLStatusStore` then read via provider

### Unit tests (no compose stack)

- Pattern: `InMemoryStatusStore` for status; `FakeLiveGraphClient` for Neo4j
- Cover: provider construction, dict shape, error paths
